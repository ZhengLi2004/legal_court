# 实验设置与评估方案规划 (Experimental Setup)

## 1. 实验目标与验证对象

本实验旨在通过端到端（End-to-End）系统评测与控制变量分析，验证以下核心主张：

- **主张 1（端到端整体有效性）：** 论证图架构在包含“动态诉求生成”与“对抗推理”的全链路任务上，相较于基线模型具有更优的综合表现。_(Primary Metric: Status Accuracy (Matched) 及 End-to-End Status Accuracy)_
    
- **主张 2（一致性约束与防幻觉）：** 图机制能在拓扑结构层面提供一致性保障，降低输出冲突的发生率；同时，系统在多智能体辩论中能够保持对案卷事实的综合忠实度。_(Primary Metric: 输出冲突率、总体忠实度标量 Overall Faithfulness)_
    
- **主张 3（推理能力进化与归因）：** 记忆机制能跨案由重用对抗策略，系统状态准确率的提升主要源于多轮推理能力的演进，而非单纯的诉求抽取表现或外部检索的改善。_(Primary Metric: 纯推理基准的 Status Accuracy 曲线)_
    
- **主张 4（反事实边际增益与 Pareto 平衡）：** $\Delta\Phi_t$ 机制能有效识别边际增益的衰减，在模型准确率与算力消耗间取得良好的 Pareto 平衡。_(Primary Metric: Pareto 前沿、反事实标签翻转率_ $\Delta_{root\_flip}$_)_
    

为规范展示与检验流程，实验为每个主张锁定相应的 Primary Metric，主文图表将聚焦呈现核心指标（每项主张最多 2 个数据维或 1 张核心图）；各主张独立的检验族实施 Holm-Bonferroni 多重比较校正，其余所有解释性、敏感性与诊断性指标将作为稳健性证据统一置于附录，以确保主叙事逻辑的收敛与清晰。

## 2. 数据划分与端到端联合对齐

针对动态推演中存在的对齐与状态模糊问题，实验确立以下标准化的评测协议：

- **规范的数据集隔离与分层重采样 (Data Separation & Stratified Resampling):**
    
    - 从总样本中独立划分 50 个案件作为开发集（Dev Set），剩余作为测试集。先按（案由 $\times$ 单/多诉求 $\times$ 文本长度桶）完成切分；再在开发集内部按（案由 $\times$ 单/多诉求 $\times$ 文本长度桶 $\times$ Dev 预标签难例）进行 5 次严格分层重采样，联合层配额与 Dev 基线保持完全一致（比例不变，`TV=0`）。最终参数取 5 次重采样结果的中位数，稳定区间定义为 [20%, 80%] 分位点。
        
    - **重采样收敛证据:** 附录将提供一张“稳定性证据图”，展示参数区间宽度、主指标均值与 CI 随重采样次数（1至5次）的变化趋势，以辅助论证样本规模与参数估计的收敛性。
        
    - **难例标签的特征分离与稳健性:** 难例标签来源于输入侧可得特征或纯规则脚本。分层策略与阈值仅在开发集上确定并冻结；全量样本仅按同一冻结规则打标用于分层抽样评估与难例统计。附录将公开规则清单，并执行轻量扰动测试验证区间抗扰动能力。
        
    - **调参优先级的逻辑闭环:** 调参首要目标为最优化 Step A 的抽取 F1，此举旨在稳定后续的对齐与匹配边界、减少端到端评测的结构性噪声；但最终判断系统推理优劣的准绳，仍以主表报告的 Matched / E2E Status Accuracy 为准。在抽取 F1 近似等价的区间内，再选取使事实忠实度覆盖率最高/不确定率最低的参数。
        
- **三态标签映射与评测口径 (Label Space Protocol):**

    - **Gold 原生标签层：** Claim 级 Gold Status 先以 `ACCEPTED / REJECTED / UNMENTIONED` 冻结，再映射为评测态 `VALIDATED / DEFEATED / HYPOTHETICAL`；其中 `UNMENTIONED` 同时覆盖“法院未明确处理”与“仅部分支持 / 金额调减 / 范围缩窄 / 仅支持子项”等情形。
    
    - **主评测 (2-Class):** 评测态中的 `HYPOTHETICAL` 默认折算为 `DEFEATED`（遵循法律场景中“证据不足即驳回”的防守原则）。
        
    - **辅助评测 (3-Class):** 额外报告保留评测三态的 Balanced Accuracy 或 Macro-F1 置于附表，以评估细粒度识别能力。
        
- **基线无偏化与匹配协议 (Unbiased Matching Protocol):**
    
    - **统一预测端预去重:** 在确定聚类阈值后，对所有对比方法的预测 Claim 集合统一应用层次聚类（Cosine Similarity）进行冗余合并，簇代表取距簇中心最近的 Claim。
        
    - **一对一二分图匹配与稳健性硬门槛:** 基于匈牙利算法进行映射，代价函数综合考量语义相似度与文本长度差异惩罚。正文设立并报告预注册稳健性通过阈值：定义排序基准为 `rank(method) = mean_dev(PrimaryMetric(method))`，在采用替代特征及阈值扰动区间内，各方法在 Dev 集上的排序需满足 Spearman $\rho \ge 0.8$，且基于此定义的两两成对排序翻转率 $\le 10\%$。
        
    - **失败响应:** 若该门槛未通过，则视为匹配协议未冻结成功，主张 1 的正式 Test 比较应报错中止，不进入主文结论宣告。附录仅保留失败摘要、扰动通过率、主要翻转来源及误差剖析，不以“降级结果”替代正式主结论。
        
- **Gold Claim / Gold Status 生成与人工锚点集校准：**
    
    - Gold Claim 采用“诉请块定位 + 层级结构解析 + 规则归一 + 专家人工裁决”的协议，仅从 `原告诉称` 抽取；Gold Status 在冻结后的 Gold Claim 上采用“规则优先 + 检索/重排 + 零样本补判 + 专家人工裁决”的协议，不以生成式 Judge 作为主流程。
        
    - 如资源允许，可额外构建联合双盲人审锚点集，并固定到同一批 case 上同时完成 Claim 边界复核与 Status 标签复核；Claim 侧报告 case-level claim-set Cohen's $\kappa$，并同时报告 Exact Match 与 Claim Precision / Recall / F1；Status 侧仅在 matched claims 上计算并报告 Cohen's $\kappa$ 与 agreement rate。若因人力或周期限制未执行，则在实验手册与附录中留白并注明“未执行”，不作为当前主实验阻塞项。

## 4. 主检验指标与评估定义 (Primary Metrics & Formal Definitions)

### 4.1 主张 1：分步可解释的微观 F1 与端到端闭环

主表聚焦呈现核心闭环表现，分步指标置于附录：

- **统一聚合口径 (Case-level Macro-average):** 所有 Accuracy、Err 或 Flip 相关的聚合，遵循：先在 Case 内对所有 Root Claim 计算均值得到 Case Score，再跨 Case 求均值并基于 Paired Bootstrap 计算 95% CI。
    
- **主表核心指标:**
    
    - **E2E Status Acc:** 端到端状态准确率，未匹配抽取的 Gold Claim 直接计为状态错判（主结论首要依赖项，以反映全链路任务效用）。
        
    - **Status Acc (Matched):** 在成功匹配的样本子集上的状态准确率。解释边界：该指标定性为“条件化推演质量（给定抽取成功的前提下）”，用于解耦推演模块的内部表现，但不单独用于宣告端到端有效性。
        
- **附录分解与对照指标:**
    
    - **Step A (抽取表现):** Claim 匹配层面的 Precision / Recall / F1。
        
    - **E2E F1 (FP-Sensitive):** 显式惩罚“过度生成”或“漏抽”的双罚指标（匹配且状态错判计 FP+FN；未匹配 Gold 计 FN；无法匹配的预测计 FP）。
        
    - **指标定位与写作纪律:** 声明主结论在 E2E Status Acc 与 Soft 版本（状态错判仅计为 FN）下方向一致；在主文撰写时，主张1结论段需显式引用该结果：指出附录报告的 FP-sensitive E2E F1 与过度生成率指标，证实主结论并不依赖于过度生成（FP）偏差。
        

### 4.2 主张 2：一致性检查与事实忠实度

- **异质双层评测器配置与离散标签声明:** 涉及一致性推断、证据对齐与 NLI 的评测均采用确定性的离线管线。**主评测集合设定为异质模型组合：包含 1 个大型指令微调模型（如 Qwen2.5-72B-Instruct）、1 个不同模型家族的代表作，以及 1 个明确的 NLI/判别专长模型**，以切实减少单一评测偏置；商用补充集合（如 GPT-4o）仅用于附录交叉验证，并严格记录调用日期与 Endpoint 版本号。
    
    - **一致率动机与硬门槛:** 所有评测器固定系统提示词，强制输出 `{E, N, C}` 离散标签，不依赖置信度分数。预注册一致性硬门槛设为多评测器一致率不低于 70%（该阈值源于 Dev 集经验分布，低于此值不确定率通常显著上升）。**为支撑异质评测的可信度，附录将提供模型两两一致率矩阵、总体多数票一致率及不确定率**，并验证阈值在 $\pm 5\%$ 扰动下主要结论方向不变。若未达门槛，则主张 2 的正式结论报错中止，附录仅保留失败日志与诊断摘要。
        
    - **可运行复现块:** 附录提供高可执行性的复现清单，涵盖权重快照、推理框架、量化策略、硬件配置、Batch/上下文长度，并补充运行命令模板、随机种子策略及配置路径 Tag。
        
- **跨方法统一的一致性解析管线 (Unified Consistency Parsing):** 为缓解基线模型因缺乏显式结构而呈现的天然评测偏差，对所有方法的输出统一采用离线解析器生成 `Claim -> {status set}` 映射并推断冲突边。
    
    - **共享归并阈值与双任务敏感性图表:** 优先应用可解释规则（法条号、关键实体等）进行同义归并，Embedding 相似度作为补充。该相似度阈值默认与预测端匹配阈值共享。主文声明：共享阈值旨在减少自由度。**附录将提供一张综合敏感性图表，统筹展示阈值在** $\pm\delta$ **扰动下的（i）Merge-Error、（ii）冲突率上下界以及（iii）抽取 F1/主指标的变化**，以证实跨任务定义粒度下的趋势平稳性。
        
    - **归并审计与解析失败分布:** 解析失败计入 Parse-Failure，并在主表报告解析器覆盖率。附录提供轻量级解析与归并审计摘要：列出不可解析原因的分布，以及过度归并率、漏归并率；同时提供基于解析失败样本的冲突率 Upper/Lower Bound。
        
- **解耦的事实忠实度评估 (Decoupled Fact Faithfulness):**
    
    - **独立后处理器的证据对齐:** 评测对齐器作为独立的离线组件，Embedding 模型与阈值固定于开发集并冻结。输出 Top-m ($m=3$) 证据窗口，对齐失败单列为“对齐失败率”。
        
    - **三段式多数票 NLI 评估与聚合敏感性分析:** 在对齐成功的子集上应用异质模型判定蕴含关系，应用多数票决，平局标为“不确定”。主实验采用任一窗口蕴含即蕴含（Any）策略以保证召回；附录提供 Majority/Max-score 聚合及不确定上下界的敏感性对比。
        
    - **总体忠实度标量 (Overall Scalar):** **Overall Faithfulness = 对齐成功率** $\times$ **成功子集上的 NLI 蕴含率**。附录补充基于 $\min(\text{对齐率}, \text{蕴含率})$ 或均值加权的替代聚合方案作为合理性检验。
        

### 4.3 主张 3：推理归因的冻结输入 Benchmark

在线增量学习实验引入纯推理主指标及固定证据包对照：

- **测试集零更新声明:** 记忆库与推演策略的更新仅在训练或开发集流程中进行。测试集阶段仅执行评估，不触发任何记忆写入、提示词自适应或检索缓存学习。测试集推演按案件时间序列单向前进且不回看；若案件时间戳不可靠，则依据数据集固定顺序严格单向推进。
    
- **固定证据包对照:** 旁路外部检索 API，系统仅读取缓存的固定证据包。测试在仅存推演策略变化时的 Status Accuracy 演化，辅助归因推理能力的进步。
    

### 4.4 主张 4：受控反事实边际增益 (Controlled Counterfactual Gain)

- **对称预算控制与 Pareto 构造规则:** 所有对比方法在相同的算力上限网格下运行（裁剪协议：触达上限时仅裁剪推演过程，维持合规的任务输出 Schema）。
    
    - **环境锁定与噪声容忍阈值:** 每个预算点在 Temperature=0 设定下执行 3 次独立重复运行。独立性限定为：在固定检索索引、并发度及工具版本的前提下，仅反映外部 API 返回的不可控波动与系统调度造成的微小非确定性。**明确预注册噪声容忍阈值：要求各预算点上的标准差** $\le$ **方法间均值差距的 1/3**（旨在防止运行噪声达到或超过效应量，导致评估失效）。附录将单独报告各运行间的方差估计；**若任一预算点超阈，则主张 4 的正式比较报错中止，并保留失败日志与噪声审计结果；不采用“剔除该点后继续宣告”作为兜底方案。** Pareto 前沿仅在全部预算点通过硬门槛后构造。
        
- **受控重算与多视角分析:** 在预设白名单缓存（仅复用原始检索证据）下重算。
    
    - **主指标 (Root 视角):** $\Delta_{root\_flip}$ 作为主图呈现。
        
    - **解释性附图:** $\Delta_{err}$ 与 $\Delta_{conflict}$ 作为补充视角置于附录。
        

## 5. 统计呈现收敛与评测协议冻结 (Statistical Presentation & Protocol Freeze)

为防范“显著性检验泛滥”及潜藏的实验偏置，确立以下保障策略：

- **主文呈现纪律:** 主表与主图统一采用 **95% CI (基于 Case-level Paired Bootstrap)** 呈现方差，并仅在满足 Holm-Bonferroni 多重比较校正的**预注册主点**上标注显著性星号。对于预注册点之外的探索性节点与完整曲线，一律只报 CI，不标注星号，不做显著性语言宣告。
    
- **附录补充检验:** **McNemar 检验**（用于 `(case, root-claim)` 配对的 2-Class 状态差异）与 **Stuart-Maxwell 检验**（用于 3-Class 辅助评测）的结果全表，统一归入附录供详查。
    
- **预注册比较点声明:** 针对主张 4 的显著性检验，预注册范围限定于“主系统 vs 各基线”的配对比较。预算网格选取 3 个分位阈值点（如 Token 上限的 25%、50%、75%），回合选取 $t \in \{1, 3, 5\}$，通过预定检验族规模控制假阳性率。
    
- **评测管线全局冻结声明 (Global Freeze Commitment):** 实验承诺：所有评测组件在启动测试集 (Test Set) 评估前冻结，并在代码库中执行 Tag 归档。可审计的归档条目路径包含：模型配置 (`configs/*.yaml`)、解析器规则 (`parser_rules/*.json`)、对齐器参数 (`aligner/*.yaml`)、对称预算网格 (`budget_grid.json`) 与预注册检验点列表 (`prereg_points.json`)，以支持评测方法学的透明度与独立审计。
